### Q3. 编程语言选什么？

| 选项 | 优势 | 劣势 |
|------|------|------|
| TypeScript/Node.js | 生态丰富、npm 分发成熟、JSON 处理自然 | 异步模型复杂、单文件二进制分发困难 |

### Q8. 重点深入维度：你选哪个维度做深？

文档要求六个维度都有最低实现，但**选一个做深入**。建议选项：

- **治理（护栏/沙箱/HITL 状态机/范围围栏）**：最"代码化"的维度，天然适合写确定性测试
- **反馈闭环（确定性校验器 + 失败分类 + 多轮自我修正）**：同样天然是代码，工程深度好
- **扩展（工具分发/多 agent 编排）**：如果你想让多个 agent 协作
- **记忆/上下文工程**：存储与检索必须自己实现，不能直接用框架的 memory

你的选择及理由？
